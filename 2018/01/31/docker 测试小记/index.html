Docker 测试小记

发表于 2018-01-31

Docker Test

这里记录对 Docker 的测试。

需要说明的是，领导对 docker 的理解是轻量的虚拟机，所以总是想着当虚拟机来使用。这次的任务就是用 docker 跑 Windows 图形界面的应用。坑挖好了，请你跳了。

  • windows10 docker 安装
  • 方案文档

方案倾向于在Windows下使用docker技术，做轻量化的虚拟机，能够支持Windows系统，也能支持Linux系统。

但是windows下docker使用主要是用来跑gui工具，这方面不是docker强项，因此也是坑比较多。

最后也把 Windows 方案抛弃了，采用 Linux 上跑 docker，不是跑服务，而是跑一些图形化工具。

test list

  • Windows下测试，测试不通过，坑太多，转换方案
  • Linux 虚拟机上测试，要求能够启动图形界面工具
  • 解决字体显示问题，Linux下的字体全部拷贝进去了，需要精简，最后测试需要 fonts/vlgothic
  • 工程目录映射出来
  • Linux实体机上运行docker
  • 私有仓库建立
    • docker run registry
    • 仓库管理系统
      • humpback，国内的运行了，images显示不出来，管理太简单了
      • harbor，下载了最新版本，但是要求docker版本比较高，而高版本的docker需要redhat7.4版本，版本问题需要处理一下，了解harbor哪个版本能支持 docker v1.12

test1 windows10 下测试 docker

首先按方案在Windows10下测试，有各种问题

  • 容器切换，支持Windows容器和Linux容器，需要切换，pull的image要分清是Linux还是Windows的，否则运行不了。运行时加—isolation=hyperv来选择在hyper-v运行
  • 容器代理，经常配置了运行不了，这块太不稳定
  • 镜像文件大小，选择docker也是为了镜像文件比虚拟机小，但是也是以G算

windows 的镜像文件及大小

  • 在microsoft/windowsservercore容器里开启远程服务：net start termservice，也无法在外部通过远程连接，内部看不到端口3389

test2 Linux虚拟机上测试

Windows上测不通，转方案，在Linux虚拟机上测试，要求能用docker运行图形界面工具。

docker 升级

升级到1.17版本，运行有错误

这个问题是docker1.17版本要求Linux是7.4版本, 懒得升级操作系统了，把 docker 降到1.12版本
